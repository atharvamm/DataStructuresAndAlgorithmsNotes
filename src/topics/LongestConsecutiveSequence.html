Longest Consecutive Sequence

Approach

The question is asking us to find the length of longest consecutive sequence. The meaning of consecutive means the difference between the elements will be 1. Also the array is unsorted and we have to solve the problem in O(n) time.

Given that the array is unsorted and we have to solve in O(n) time, we can understand that we are not allowed to sort (Context: Sorting takes nlogn time.). Given the fact the only we can solve the problem is if we know for a given number is the previous number or the next number is in the array or not.

For a number if we know that the next number is in the array we can keep doing until we reach a point where the next number is not in the array. If we keep track of the length till we reach this number, we will also get the length of max consecutive elements.

To further optimize our solution, what we can do is only start at numbers which are the highest/largest in the chain of consecutive numbers. By doing this we wont have to find sub-chains of the longest sequence.

Main Ideas

  • Whenever we face a problem where we might need an event or value later, store the value in a dictionary/set so that you will not have to iterate through the array to see if the event happened in the past.
  • When working on sequence problems note where the sequence starts and where it ends, this will help eliminate repetitve work, where you accidentally traverse a sub-sequence.
