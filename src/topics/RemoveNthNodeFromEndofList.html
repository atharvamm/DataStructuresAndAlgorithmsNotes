Remove Nth Node From End of List

Approach

Two approaches here:

  • Go through the linked list and store the elements in an array, then when we store the last element go back and check the element at nth position from last. Return the element at index 0. You can also add checks here to stop doing this process if there is just on element in the list and n = 1.
  • An easier method would be to have two pointers. Once pointer starting at dummy and the other at dummy+n. Now we keep moving the second pointer until it reaches the last element. Then when it reaches the last element, we just take the left pointer and changes it next to next of next element.

Main Ideas

  • For linked list problems see if adding a dummy node in the beginning helps. If you added a dummy node and the problem requires you to return the head of linked list, return dummy.next and not head.
  • If you need to find the end of linked list in one pass in your problem and then do some operation, you can use two pointers. Try to create some sort of shift between the 2 pointers such that the second pointer reaches the end and then you can use that information to do some sort of operation.
